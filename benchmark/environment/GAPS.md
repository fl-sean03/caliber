# Gaps to the release gate

**Release gate:** an outside machine reproduces one full task run + grade from the
public docs alone. This page is the honest delta between the v0 draft
([`TOOLCHAIN.md`](TOOLCHAIN.md), [`Dockerfile`](Dockerfile),
[`smoke_task.md`](smoke_task.md)) and that gate. Nothing here is hidden inside
optimistic comments — if it isn't demonstrated, it's listed.

| # | Gap | Why it blocks / risks the gate | Recommended next step | Rough effort |
|---|-----|-------------------------------|------------------------|--------------|
| G0 | **The Dockerfile has never been built.** It is a faithful transcription of the reference builds, but apt runtime package names (e.g. `libopenmpi3t64`, `libopenblas0-openmp`), the NVHPC image tag, and the micromamba bootstrap (currently `latest` — itself an unpinned input) are unverified. | A stranger's first command fails. | Iterate `docker build` to green on a clean machine; pin the micromamba version; record final image digests. | 0.5–1 day |
| G1 | **QE build is not hermetic.** QE's CMake fetches three external submodules at configure time (wannier90, libmbd, devicexlib) with no checksum pinning; upstream moving/rehosting breaks the build or silently changes inputs. Pseudopotentials are also fetched at *task runtime* (agents download UPF files), unpinned. | Reproducibility of both the build and the physics inputs depends on third-party servers. | Vendor the three submodule snapshots (or pre-populate the source tree in a cached layer) with sha256s; ship a checksummed pseudopotential library for the task slate inside the image (SSSP/pslibrary subsets are redistributable — verify per-file licenses). | 1–2 days |
| G2 | **LAMMPS tarball has no upstream sha256.** Pinned by git tag only. | Weaker integrity anchor than QE's checksummed tarball. | Compute and commit the tarball sha256 on first green build (tag→tarball is stable on GitHub). | minutes |
| G3 | **MLIP weights are downloaded at first use, unchecksummed.** MACE-MP-0 pulls from a GitHub release into a cache dir; matgl pulls `M3GNet-PES-MatPES-PBE-2025.2` from Hugging Face (upstream has already retired a checkpoint once — the MP-2021 model vanished in matgl 4.x); CHGNet bundles weights in the wheel. Licenses for *redistributing* baked weights (MIT/BSD code, but weights trained on Materials Project data carry CC-BY-4.0 attribution requirements) are unaudited. | An offline or rate-limited container can't run MLIP tasks; a silent upstream weight swap changes the physics under the same version pins; redistribution without a license audit is a legal risk. | Bake the exact weight files into the image with recorded sha256s, set the libraries' offline/cache env vars, and add a NOTICE file with per-model licenses + attributions. If redistribution is disallowed for any model, ship a checksum-verified first-run fetch script instead. | 1 day + license review |
| G4 | **Harness ergonomics are still reference-machine-shaped.** The private root is now env-configurable (`CALIBER_PRIVATE`, default `~/.caliber-private`) — but the sweep still reads a fleet-account pin file and embeds a reference-machine preamble ("RTX 5080, 24 cores", `source ~/miniconda3/...`), and the smoke protocol still hand-builds the goal file. Workable, but not stranger-grade ergonomics. | The gate says "from README alone"; today the README needs a workaround section. | Add a `--preamble` variant for container runs and make the account-pin file optional; keep defaults byte-compatible with the reference machine so existing runs regrade identically. | 0.5 day + regrade spot-check |
| G5 | **GPU-in-container portability and size.** All GPU binaries are `sm_120` (Blackwell) on the reference machine; a stranger's A100/H100 needs a rebuild (`--build-arg GPU_ARCH=...`) rather than a pull. The optional NVHPC runtime copy for GPU QE is multi-GB and un-slimmed. Cross-arch numerical variance is argued (TOOLCHAIN.md) but not yet *measured* beyond the reference CPU/GPU cross-check. | "Pull and run" fails on non-Blackwell GPUs without a rebuild; nobody has demonstrated anchor stability across GPU generations. | Build LAMMPS GPU as a fat binary for {sm_80, sm_90, sm_120}; keep GPU QE behind the optional target; run the smoke task once on a rented A100/H100 and diff reported values against reference (this doubles as the outside-machine gate run). | 1–2 days incl. one cloud run |
| G6 | **Sealed-task grading for outsiders is unsolved.** The stranger test grades a public smoke key; real Caliber tasks require the sealed keys in `caliber-private`. There is currently no mechanism for an outside lab to grade a real run without us. | The gate is met for *one task* via the smoke key, but the benchmark-as-product needs a story: nobody trusts a leaderboard only the authors can score. | Owner decision between (a) a grading service (submit `reported_values.json` + transcript digest, receive `score.json`), (b) publishing hash commitments of keys now + revealing on freeze, or (c) partial key release for a public dev split. (a) and (b) compose. Prototype (a) as a single endpoint. | design: owner call; service prototype ~1 wk |
| G7 | **Agent-CLI auth and cost accounting differ off-reference.** The container authenticates `claude` via `ANTHROPIC_API_KEY`; reference runs used account config-dir pins. Cost fields in the stream (`total_cost_usd`) and rate-limit behavior can differ between API-key and subscription auth, and the CLI itself moves fast (pinned 2.1.207). | The cost axis is a first-class score; if a stranger's cost accounting isn't comparable, their reproduction isn't a reproduction of the score. | Document API-key auth as the canonical stranger path; verify `total_cost_usd` parity on one API-key run vs a reference run; treat CLI version as recorded harness provenance (it already is: `harness:{name,version}`). | 0.5 day |
| G8 | **Cloud image strategy undecided.** A ~20–40 GB image needs a registry home (GHCR?) and a target platform. Vast.ai is under reconsideration for reliability; serverless (e.g. Modal) images are built from their own toolchain, not a pulled Dockerfile, so this Dockerfile would be *transcribed*, not reused, there. | Outside reproduction on rented GPUs is the realistic path for most strangers; without a published image they must build for an hour first. | Publish the built image to GHCR with a digest pin in the README; decide the rented-GPU reference platform; if serverless is chosen, generate its image spec from TOOLCHAIN.md so there is one source of truth. | 1 day + owner platform call |
| G9 | **The frozen judge is an external, mortal dependency.** Grading needs OpenAI's `gpt-5.5-2026-04-23` snapshot to stay served; deprecation would strand exact-regrade reproducibility (mechanical anchors — the gate — are unaffected). | Long-horizon regrades of the judge axis become impossible without a migration protocol. | Persist full judge transcripts (prompt + verdict) per run so historical scores remain auditable; pre-write the judge-migration/recalibration protocol (kappa check against the human gold set) before it's needed. | 0.5 day |

## Suggested order of attack

1. **G0 → G2** (make it build, checksum everything) — everything else is theoretical
   until the image is green.
2. **G4** (private-root/preamble parameterization) — smallest change with the biggest
   ergonomics payoff for the stranger protocol.
3. **G3** (weights escrow + licenses) — required for any offline/cloud run and for
   physics stability under fixed pins.
4. **G5** (one paid outside-machine run) — this run *is* the release gate; do it as
   soon as G0–G4 land.
5. **G6/G8** need owner decisions before engineering; **G7/G9** are half-day
   hardening items that can ride along.
